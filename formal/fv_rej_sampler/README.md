# rej_sampler_ctrl
Date: 26.03.2025
Author: LUBIS EDA GmbH

## Folder Structure
The following files are part of the main directory **formal/fv_rej_sampler**:

- **fv_rej_sampler_top.sv**: Top-level assertion IP (AIP) containing the properties
- **fv_rej_sampler_constraints.sv**: Constraints for formal verification
- **fv_rej_sampler_scoreboard.sv**: Scoreboard utility that checks data integrity and ordering of the buffer used in **rej_sampler**


## DUT Overview

The DUT **rej_sampler_ctrl** has the primary inputs and primary outputs as shown below.

| S.No | Port              | Direction | Description                                                                            |
| ---- | ----------------- | --------- | ---------------------------------------------------------------------------------------|
| 1    | clk               | input     | The positive edge of the clk is used for all the signals                                  |
| 2    | rst_b             | input     | The reset signal is active low and resets the core                                        |
| 3    | zeroize           | input     | The core is reseted when this signal is triggered.                                        |
| 4    | data_valid_i    | input     |  Handshaking singal to tell that the input data is valid                                        |
| 5    | data_hold_o      | output     | Handshaking singal to tell that the module is not ready to take data                       | 
| 6    | data_i       | input     | Input data                                               |
| 7    | data_valid_o      | output     | Output data is available.                                            |
| 8    | data_o      | output     | Output data.                                             |

The design serves as a ctrl unit for the REJ_SAMPLER. It maintains a buffer that holds coefficients generated by the **rej_sampler**.

## Assertion IP Overview

The Assertion IP signals are bound with the respective signals in the dut. The assertions checks the behavior of the hold_o signal, and the behavior of the dut after reset. It also instantiates the scoreboard, and checks the data integrity and ordering of the buffer. 


### fv_rej_sampler_top.sv

- reset_assert: Checks if the **data_hold_o** and **data_valid_o** is low after reset.
- check_hold_o_output: checks if **data_hold_o** behaves as specified.
- The instance of scoreboard checks if the data integrity and ordering is maintained in the buffer.
